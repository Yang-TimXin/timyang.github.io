# Cursor 基础用法

## 核心功能概览

Cursor 的核心功能可以分为四大类：AI 对话、智能补全、代码编辑、项目理解。

---

## 1. AI 对话（Chat）

### 快捷键
- **打开 Chat 面板**：`Cmd/Ctrl + L`

### 基本用法

```
帮我分析这个函数的作用，并优化它的性能
```

### 高级技巧

#### @ 引用功能
在 Chat 中使用 `@` 可以引用特定内容：

| 引用方式 | 说明 |
|---------|------|
| `@Code` | 引用选中的代码 |
| `@File` | 引用整个文件 |
| `@Folder` | 引用整个文件夹 |
| `@Web` | 搜索网页内容 |
| `@Docs` | 引用官方文档 |
| `@Codebase` | 搜索整个代码库 |

#### 示例

```
@Code 这段代码有什么问题？帮我修复
```

```
@File config.ts 帮我解释这个配置文件的每个选项
```

```
@Web React 18 的新特性是什么？帮我更新这个组件
```

## 2. 智能补全（Tab）

### 快捷键
- **接受补全**：`Tab`
- **拒绝补全**：`Esc`

### 使用场景

#### 场景一：写新函数
```python
def calculate_average(numbers):
    # 输入这里，Cursor 会自动补全
```

#### 场景二：根据注释生成代码
```javascript
// 创建一个函数，接收用户对象数组，返回成年用户
// 输入后按 Tab 接受补全
```

#### 场景三：补全测试用例
```python
def test_add():
    assert add(1, 2) == 3
    # Cursor 会自动补全更多测试用例
```

### 提高补全质量

1. **写清晰的注释**：注释越清楚，补全越准确
2. **保持上下文**：让 Cursor 看到相关的代码
3. **使用类型注解**：TypeScript/Python 类型注解帮助理解

## 3. 内联编辑（Inline Edit）

### 快捷键
- **打开内联编辑**：`Cmd/Ctrl + K`

### 使用场景

#### 场景一：重构代码
1. 选中代码
2. 按 `Cmd/Ctrl + K`
3. 输入指令：`提取为独立函数`

#### 场景二：添加功能
```javascript
// 选中这个函数
function processUser(user) {
    return user.name;
}

// 按 Cmd/Ctrl + K，输入：
// 添加邮箱验证，如果邮箱无效抛出错误
```

#### 场景三：修复 Bug
```python
# 选中有问题的代码
result = data['key']

# 按 Cmd/Ctrl + K，输入：
// 添加 KeyError 处理
```

## 4. Composer（多文件编辑）

### 快捷键
- **打开 Composer**：`Cmd/Ctrl + I`

### 使用场景

#### 场景一：创建新功能
```
创建一个用户认证模块，包含：
1. 登录函数
2. 注册函数
3. 密码重置函数
4. 相关类型定义
```

#### 场景二：重构项目
```
将这个项目从 JavaScript 迁移到 TypeScript：
1. 重命名所有 .js 文件为 .ts
2. 添加类型注解
3. 更新配置文件
```

#### 场景三：批量修改
```
在所有 API 路由文件中：
1. 添加错误处理中间件
2. 统一响应格式
3. 添加请求日志
```

### 高级技巧

#### 使用 @ 引用
```
@folder src/components 
将所有组件从 class 组件改为函数组件
```

#### 分步执行
```
第一步：分析现有代码结构
第二步：制定迁移计划
第三步：执行迁移
```

## 5. 项目理解

### 索引代码库
Cursor 会自动索引你的项目，但你可以手动触发：

1. 打开 Command Palette：`Cmd/Ctrl + Shift + P`
2. 输入 `Cursor: Reindex Codebase`

### 使用场景

#### 场景一：理解新项目
```
@Codebase 这个项目是做什么的？主要架构是什么？
```

#### 场景二：查找相关代码
```
@Codebase 找到所有处理用户认证的代码
```

#### 场景三：分析依赖
```
@Codebase 分析这个模块的所有依赖关系
```

## 6. 终端集成

### 快捷键
- **打开终端**：`` Ctrl + ` ``
- **在终端中使用 AI**：选中终端输出，按 `Cmd/Ctrl + L`

### 使用场景

#### 场景一：解释错误信息
```bash
# 运行命令后出现错误
$ npm run build
Error: Module not found

# 选中错误信息，按 Cmd/Ctrl + L
# 输入：解释这个错误并给出解决方案
```

#### 场景二：生成命令
```
帮我写一个 find 命令，查找所有大于 100MB 的文件
```

#### 场景三：自动化脚本
```
帮我写一个 shell 脚本，自动备份数据库并压缩
```

## 实用技巧

### 1. 快捷键速查表

| 功能 | 快捷键 |
|------|--------|
| Chat 面板 | `Cmd/Ctrl + L` |
| 内联编辑 | `Cmd/Ctrl + K` |
| Composer | `Cmd/Ctrl + I` |
| 接受补全 | `Tab` |
| 拒绝补全 | `Esc` |
| Command Palette | `Cmd/Ctrl + Shift + P` |

### 2. 提高效率的技巧

1. **组合使用**：Chat + Inline Edit + Composer
2. **上下文优先**：先让 Cursor 理解代码，再让它修改
3. **分步执行**：复杂任务拆分成小步骤
4. **验证结果**：每次修改后都要测试

### 3. 常见错误避免

| 错误 | 正确做法 |
|------|---------|
| 直接让 AI 重写整个文件 | 分步骤修改 |
| 不测试就提交 | 每次修改后测试 |
| 忽略 AI 的建议 | 仔细审查 AI 的修改 |
