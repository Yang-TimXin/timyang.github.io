## 3. 高级功能

### 代码库索引优化

#### 手动索引
```bash
# 在终端中运行
cursor --reindex
```

#### 配置索引规则
在项目根目录创建 `.cursorignore`：
```
node_modules
dist
build
.git
*.log
```

### 自定义规则

在项目根目录创建 `.cursorrules`：
```markdown
# 项目规范

## 代码风格
- 使用 TypeScript strict 模式
- 遵循 ESLint 规则
- 使用 Prettier 格式化

## 命名规范
- 变量：camelCase
- 函数：camelCase
- 类：PascalCase
- 常量：UPPER_SNAKE_CASE

## 文件结构
- 组件：src/components/
- 工具函数：src/utils/
- 类型定义：src/types/
```

### 快捷命令

在 Chat 中使用 `/` 命令：

| 命令 | 说明 |
|------|------|
| `/explain` | 解释代码 |
| `/fix` | 修复问题 |
| `/refactor` | 重构代码 |
| `/test` | 生成测试 |
| `/docs` | 生成文档 |

## 4. 性能优化

### 减少 Token 消耗

#### 技巧一：精确引用
```
# 不好
看看这个文件有什么问题

# 好
@File src/utils/auth.ts 检查这个文件的错误处理
```

#### 技巧二：分步执行
```
# 不好
重写整个模块

# 好
第一步：分析现有代码
第二步：设计新架构
第三步：逐步实现
```

#### 技巧三：设置上下文
```
这是一个 React 项目，使用 TypeScript 和 Redux。
请帮我优化这个组件的性能。
```

### 提高响应速度

1. **关闭不必要的扩展**
2. **定期清理索引**
3. **使用 `.cursorignore` 排除无关文件**

## 6. 调试技巧

### 常见问题解决

#### 问题一：AI 不理解上下文
**解决方案：**
1. 使用 `@Code` 或 `@File` 明确引用
2. 在提示词中提供更多背景信息
3. 分步引导 AI 理解

#### 问题二：生成的代码有问题
**解决方案：**
1. 使用 `/fix` 命令
2. 提供更详细的描述
3. 分步骤实现

#### 问题三：性能下降
**解决方案：**
1. 配置 `.cursorignore`
2. 清理索引缓存
3. 关闭不必要的功能

## 7. 进阶用法

### 自定义快捷键

在 `keybindings.json` 中添加：
```json
[
  {
    "key": "cmd+shift+l",
    "command": "cursor.chat"
  },
  {
    "key": "cmd+shift+k",
    "command": "cursor.inlineEdit"
  }
]
```

### 脚本自动化

创建 `scripts/cursor-helper.sh`：
```bash
#!/bin/bash
# 快速打开 Cursor 并打开指定文件
cursor --goto "$1:$2"
```
