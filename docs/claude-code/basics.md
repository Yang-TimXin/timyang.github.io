# 基础用法

## 启动 Claude Code

```bash
# 在项目目录启动
cd your-project
claude
```

启动后你会看到一个交互式终端界面，可以直接开始对话。

## 核心命令

### 对话命令

| 命令 | 说明 |
|------|------|
| 直接输入 | 发送消息给 Claude |
| `/help` | 查看帮助信息 |
| `/clear` | 清空当前对话 |
| `/exit` | 退出 Claude Code |

### 快捷操作

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + C` | 取消当前操作 |
| `Ctrl + L` | 清屏 |
| `Tab` | 接受建议 |

## 常用场景

### 1. 分析项目结构

```
> 帮我分析这个项目的目录结构，告诉我每个文件夹的作用
```

### 2. 编写新功能

```
> 帮我写一个 Express 路由，处理用户注册，需要邮箱验证
```

### 3. 调试 Bug

```
> 这个函数报错了：TypeError: Cannot read property 'map' of undefined
帮我找出问题并修复
```

### 4. 代码重构

```
> 帮我把这个 class 组件重构成 hooks 函数组件
```

### 5. 编写测试

```
> 为 UserService 类编写单元测试，覆盖所有公共方法
```

## 使用技巧

### 提供上下文

好的提示词应该包含足够的上下文：

```
# ❌ 不好的提示
帮我写个函数

# ✅ 好的提示
在 src/utils/date.js 中添加一个 formatDate 函数，
接收 Date 对象，返回 "YYYY-MM-DD HH:mm:ss" 格式的字符串
```

### 分步骤描述

复杂任务拆分成小步骤：

```
我需要实现一个用户登录功能：
1. 创建登录表单组件
2. 添加表单验证逻辑
3. 对接后端 API
4. 处理成功/失败响应
请先帮我完成第一步
```

### 利用上下文

Claude Code 会读取你的项目文件，所以可以引用具体文件：

```
看看 src/api/user.js 中的 getUser 函数，
帮我添加错误处理和重试机制
```
