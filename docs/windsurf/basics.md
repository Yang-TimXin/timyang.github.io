# Windsurf 基础用法

## 核心功能概览

Windsurf 的核心功能包括：Cascade 智能生成、AI 对话、代码补全、项目理解。

---

## 1. Cascade（智能生成）

### 快捷键
- **触发 Cascade**：在编辑器中直接输入或按 `Tab`

### 基本用法

#### 场景一：根据注释生成代码
```python
# 创建一个函数，计算斐波那契数列的第 n 项
# 输入注释后，Windsurf 会自动生成代码
```

#### 场景二：补全代码
```javascript
function processUser(user) {
    // 输入这里，Windsurf 会根据上下文补全
}
```

#### 场景三：生成测试
```python
def add(a, b):
    return a + b

# 在下方输入注释：
# 测试 add 函数的各种情况
# Windsurf 会自动生成测试用例
```

### Cascade 的优势

| 特性 | 说明 |
|------|------|
| **上下文感知** | 理解整个项目结构 |
| **实时生成** | 边写边生成，无需等待 |
| **智能补全** | 不只是补全单词，而是补全逻辑 |
| **多文件理解** | 能跨文件理解依赖关系 |

## 2. AI 对话（Chat）

### 快捷键
- **打开 Chat 面板**：`Cmd/Ctrl + L`

### 基本用法

```
帮我优化这个函数的性能
```

### 高级技巧

#### @ 引用功能
在 Chat 中使用 `@` 引用内容：

| 引用方式 | 说明 |
|---------|------|
| `@Code` | 引用选中的代码 |
| `@File` | 引用整个文件 |
| `@Folder` | 引用整个文件夹 |
| `@Web` | 搜索网页内容 |
| `@Docs` | 引用官方文档 |

#### 示例

```
@Code 这段代码有什么问题？
```

```
@File src/api.ts 帮我解释这个 API 接口
```

```
@Web React 18 的新特性是什么？
```

### 对话模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **Ask** | 只回答问题 | 了解代码、学习概念 |
| **Edit** | 修改代码 | 重构、优化、修复 |
| **Add** | 添加新功能 | 开发新特性 |

## 3. 内联编辑（Inline Edit）

### 快捷键
- **打开内联编辑**：`Cmd/Ctrl + K`

### 使用场景

#### 场景一：重构代码
1. 选中代码
2. 按 `Cmd/Ctrl + K`
3. 输入指令：`提取为独立函数`

#### 场景二：添加功能
```javascript
// 选中这个函数
function processUser(user) {
    return user.name;
}

// 按 Cmd/Ctrl + K，输入：
// 添加邮箱验证，如果邮箱无效抛出错误
```

#### 场景三：修复 Bug
```python
# 选中有问题的代码
result = data['key']

// 按 Cmd/Ctrl + K，输入：
// 添加 KeyError 处理
```

### 高级用法

#### 多行编辑
```
选中多行代码后使用内联编辑，可以一次性修改多个地方
```

#### 条件编辑
```
只修改特定条件下的代码
```

## 4. Composer（多文件编辑）

### 快捷键
- **打开 Composer**：`Cmd/Ctrl + I`

### 使用场景

#### 场景一：创建新功能
```
创建一个用户认证模块，包含：
1. 登录函数
2. 注册函数
3. 密码重置函数
4. 相关类型定义
```

#### 场景二：重构项目
```
将这个项目从 JavaScript 迁移到 TypeScript：
1. 重命名所有 .js 文件为 .ts
2. 添加类型注解
3. 更新配置文件
```

#### 场景三：批量修改
```
在所有 API 路由文件中：
1. 添加错误处理中间件
2. 统一响应格式
3. 添加请求日志
```

### 高级技巧

#### 使用 @ 引用
```
@folder src/components 
将所有组件从 class 组件改为函数组件
```

#### 分步执行
```
第一步：分析现有代码结构
第二步：制定迁移计划
第三步：执行迁移
```

## 实用技巧

### 1. 快捷键速查表

| 功能 | 快捷键 |
|------|--------|
| Chat 面板 | `Cmd/Ctrl + L` |
| 内联编辑 | `Cmd/Ctrl + K` |
| Composer | `Cmd/Ctrl + I` |
| 接受补全 | `Tab` |
| 拒绝补全 | `Esc` |
| Command Palette | `Cmd/Ctrl + Shift + P` |

### 2. Cascade vs Cursor Tab

| 对比项 | Windsurf Cascade | Cursor Tab |
|--------|------------------|------------|
| 生成方式 | 实时生成 | 延迟补全 |
| 上下文理解 | 深度理解 | 基础理解 |
| 响应速度 | 极快 | 快 |
| 准确度 | 高 | 中高 |

### 3. 提高效率的技巧

1. **优先使用 Cascade**：它是 Windsurf 的核心优势
2. **组合使用**：Cascade + Chat + Inline Edit
3. **上下文优先**：先让 Windsurf 理解代码，再让它修改
4. **验证结果**：每次修改后都要测试

## 8. 常见问题

### 问题一：Cascade 不触发
**解决方案：**
1. 检查是否启用了 AI 功能
2. 重新加载窗口
3. 检查网络连接

### 问题二：生成的代码有问题
**解决方案：**
1. 使用 `/fix` 命令
2. 提供更详细的描述
3. 分步骤实现

### 问题三：性能下降
**解决方案：**
1. 清理索引缓存
2. 关闭不必要的功能
3. 重启 Windsurf
