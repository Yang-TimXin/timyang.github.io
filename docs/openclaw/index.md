# OpenClaw 全面教程

> 面向中文开发者的 OpenClaw 实战指南，从入门到高级应用。

## 什么是 OpenClaw

OpenClaw 是一个开源的个人 AI 助手框架，让你可以像私人管家一样控制自己的 AI。不同于传统的 ChatGPT 网页或 API 封装，OpenClaw 提供了一个完整的运行时环境，让你的 AI 助手具备：

- **跨平台集成**：同时接入微信、Telegram、Discord、WhatsApp 等多个消息通道
- **工具执行能力**：可以直接运行命令、操控浏览器、读写文件、搜索网页
- **记忆系统**：拥有持久化记忆，跨会话保持上下文连续性
- **插件生态**：通过插件系统扩展功能，社区共享
- **多设备协作**：支持 Android/iOS/macOS 节点配对，实现多设备联动

## 核心能力

### 🔧 工具系统

OpenClaw 内置丰富的工具：

- **文件操作**：读写、搜索、编辑本地文件
- **Shell 执行**：运行任意命令行程序
- **浏览器自动化**：通过 Chrome DevTools Protocol 控制浏览器
- **网络搜索**：集成 Brave Search、Perplexity 等搜索引擎
- **消息推送**：跨通道发送消息
- **节点控制**：远程操控配对设备

### 🧠 记忆系统

- **短期记忆**：会话内上下文
- **长期记忆**：MEMORY.md 持久化存储
- **每日日志**：memory/YYYY-MM-DD.md 自动记录
- **语义搜索**：memory_search 工具支持向量检索

### 🔌 插件架构

- 官方插件和社区插件
- 插件定义新的工具和技能
- 支持自定义 Skill（技能文件）

## 架构设计

OpenClaw 采用分层架构：

```
┌─────────────────────────────────────────┐
│              消息通道层                   │
│   微信 │ Telegram │ Discord │ WhatsApp   │
├─────────────────────────────────────────┤
│              Gateway 核心                │
│   会话管理 │ 路由分发 │ 认证授权          │
├─────────────────────────────────────────┤
│              Agent 引擎                  │
│   LLM 调用 │ 工具编排 │ 上下文管理       │
├─────────────────────────────────────────┤
│              插件/技能层                  │
│   官方插件 │ 社区插件 │ 自定义 Skill      │
├─────────────────────────────────────────┤
│              执行层                      │
│   Shell │ 浏览器 │ 文件系统 │ 网络        │
└─────────────────────────────────────────┘
```

### 关键组件

| 组件 | 说明 |
|------|------|
| **Gateway** | 核心服务，处理消息路由、会话管理、配置 |
| **Agent** | AI 推理引擎，调用 LLM 并编排工具执行 |
| **Plugins** | 插件系统，扩展工具和能力 |
| **Skills** | 技能定义，指导 Agent 完成特定任务 |
| **Browser** | 浏览器控制服务，支持 CDP 协议 |
| **Nodes** | 节点系统，支持多设备协作 |

### 工作流程

1. 用户通过消息通道发送消息
2. Gateway 接收并路由到对应 Agent
3. Agent 分析意图，决定调用哪些工具
4. 工具执行结果返回 Agent
5. Agent 综合信息生成回复
6. 回复通过原通道返回用户
